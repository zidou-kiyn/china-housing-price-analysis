# ML 模型优化：数据、训练、管理、使用（父任务）

## Goal

系统性解决当前房价预测 ML 模块在数据、训练、管理、使用四个层面的结构性问题，
让 330 城年度数据（3206 行，当前 ML 层死资产）进入训练与预测，让模型指标可信、
版本可治理、预测覆盖面显著扩大。

## 背景与现状诊断（2026-07-08 摸底）

数据库三类源：`listing_annual_58`（3206 行/330 城/年度 12 月点）、`creprice`
（396 行/13 个月/月度挂牌）、`kaggle_lianjia`（北京 82 个月/成交）。

| 层面 | 问题 |
|---|---|
| 数据 | `build_region_series` 缺失率 >30% 即跳过 → 年度数据完全进不了 ML；source/口径未作特征；挂牌/成交无校准 |
| 训练 | RF 无 CV、无 naive 基线对比、单次后 20% 切分、meta 无训练数据指纹 |
| 管理 | active 指针滞后（v1.5 活跃而 v1.6 更优）、实验版本堆积无删除/清理、无最佳版本提示 |
| 使用 | 历史 <12 个月直接拒绝 → 330 年度城市全部不可预测；XGB 置信区间用全局绝对 resid_std 对不同价位城市失真；prediction 表旧版本行残留 |

关键实证：北京双口径重叠期（2010-12~2017-12，8 对）比值从 0.79 漂移到 1.09，
**单一全局折价系数不成立**，校准须用逐年分段比值。

## 子任务地图（执行顺序）

1. `07-08-ml-dataset-builder` — 数据：多源训练集构建器（口径校准 + source 特征 + 年度扩充）
2. `07-08-ml-train-eval` — 训练：naive 基线、RF CV、分区域评估、数据指纹
3. `07-08-ml-predict-coverage` — 使用：年度城市降级预测、置信区间相对化、prediction 表治理（依赖 1）
4. `07-08-ml-model-governance` — 管理：版本删除/清理 API+UI、最佳版本提示、激活流程（独立）

依赖关系：3 依赖 1 的序列构建器；2 与 1 弱耦合（指纹依赖 1 的 dataset meta）；4 完全独立。

## 跨子任务验收标准

- [ ] 训练集能包含年度扩充样本，meta 中可见各源样本分布（死资产激活）
- [ ] 新训模型 meta 含 naive 基线对比与数据指纹，MAPE 达标线仍 ≤5%（docs/06 §6）
- [ ] 至少一个仅有年度数据的城市能返回预测（带数据质量标注），此前为 404
- [ ] 模型管理页可删除非活跃版本、可见最佳版本提示；active 指针不再滞后
- [ ] 全量后端测试通过（基线 249 passed，允许新增）；前端 build 通过

## 约束

- 遵守既有约定：源独立存储（唯一键含 source）、`source_policy.py` 唯一定义处、
  `price_select.py` 读取合并唯一入口；新增读取路径不得绕开。
- 训练目标只用月度口径价格；年度数据只作扩充（带特征标记与样本降权），不改变目标定义。
- 兼容既有模型 meta 结构：新字段只增不改，旧版本模型必须仍可加载预测。
- 每个子任务独立提交（小步提交约定）。
