# 数据管理页面 UI 精简与省份-城市树重构

## Goal

大幅精简数据管理页面，移除所有多源/已砍功能的 UI 组件，将城市列表重构为省份-城市树形结构，简化采集操作。

## Dependencies

依赖 07-08-backend-single-source 和 07-08-geo-static 完成（后端 API 变更已就绪）。

## Requirements

### 移除的 UI 组件
| 组件 | DataManageView.vue 位置 |
|---|---|
| 数据源选择框（source-card） | lines 455-489 |
| 定时采集卡片（schedule-card） | lines 519-557 |
| 数据质量卡片（quality-card） | lines 559-595 |
| "导入全国年度数据"按钮 | filter bar |
| "导入NBS 70城指数"按钮 | filter bar |
| "爬取所选地图"按钮 | batch actions |
| "补齐全部缺图"按钮 | batch actions |
| "采集全部缺数据城市"按钮 | batch actions |
| 地图状态列 | city table column |

### 保留并修改的 UI 组件
- **代理配置卡片** — 说明文字改为仅保留"代理仅作用于数据采集（creprice）。"
- **搜索框** — 合并现有两个输入框（关键词+省份）为一个，placeholder "搜索省份或城市..."，同时匹配省份名和城市名
- **"刷新城市列表"按钮** — 保留
- **"采集所选"按钮** — 唯一的采集操作按钮
- **活跃任务/历史任务区域** — 保留

### 省份-城市树形列表
- **组件**：`el-table` 树形模式（`row-key` + `tree-props`）
- **加载**：全量加载，去掉分页（330+ 城市，31 省份）
- **省份行**：显示省份名和城市计数（如"河北省 (11)"）
- **城市行列**：勾选框、城市名、城市代码、区县数、最新数据月份
- **默认状态**：全部省份折叠

### 选择功能
- **省份勾选**：勾选省份行 → 自动选中该省全部城市
- **全选**：表头勾选框 → 选中所有城市
- **单击取消**：点击已选城市的勾选框 → 取消选择
- **反选**：新增"反选"按钮，切换所有城市的选中状态

### 搜索过滤行为
- 输入省份名（如"河北"）→ 仅显示匹配的省份行
- 输入城市名（如"石家庄"）→ 显示所属省份并自动展开，仅展示匹配的城市
- 清空搜索 → 恢复完整列表（全部折叠）

### 后端 API 适配
- `GET /admin/collect/cities` 改为全量返回（去掉分页参数）
- 返回数据按省份分组（或前端自行分组）
- 去掉 `has_geo` 字段（地图已静态化，无需状态查询）

### 前端 source store
- `stores/source.ts` 中只保留 `creprice` 一个选项
- 或直接移除 source store（数据源不再可选）

### 清理
- 移除所有已删除功能相关的 script 逻辑（定时采集、数据质量、导入、地图爬取等）
- 移除前端 `api/admin.ts` 中已删除后端 API 的调用函数
- 移除 `types/index.ts` 中已删除功能的类型定义
