# 全量城市数据与地图爬取（执行型任务）

## Goal

使用管理端数据管理功能，完成 creprice 全量城市的数据采集与全部对应城市的 GeoJSON 地图爬取，使系统真正具备全国数据覆盖。本任务同时是 07-07-admin-data-collect 的实战验收。

## 前置

- 07-07-admin-data-collect 完成并合入（管理端可触发采集/爬图、任务进度可查）。
- 磁盘余量确认（raw JSON + DB 快照 + geo 文件，数百城市预估数 GB 级，执行前 `df -h` 核对）。

## 执行步骤

1. 管理端「刷新城市列表」→ 确认 city 表城市总数（预期数百）。
2. 分批触发采集（建议每批 20–50 城，串行限速），每批完成后抽查任务 result 中失败城市并记录。
3. 失败城市重试一轮；仍失败的记录原因（城市在 creprice 无数据属正常，标注即可）。
4. 触发「补齐全部缺图」地图爬取；失败城市（adcode 缺失/DataV 无边界）单独处理或标注。
5. 校验（见验收标准）并将最终覆盖统计写入本任务 notes。

## 风险与约束

- **反爬**：严格串行 + 现有限速；如触发封禁，暂停并拉长间隔，禁止调低延时换速度。
- **时长**：数百城市 × 限速间隔，预估数小时至一天，分多次执行；采集幂等（upsert），中断重跑安全。
- **数据质量**：creprice 部分小城市无区县级数据，属数据源天然限制，不视为失败。

## Acceptance Criteria

- [ ] city 表城市数 = creprice citySel 页城市总数（±数据源自身变动）。
- [ ] ≥95% 城市有区县与最新月份价格快照；失败清单及原因记录在案。
- [ ] 所有「有区县数据」的城市均有可用 GeoJSON（geo API 200），缺图清单及原因记录在案。
- [ ] 前端抽查 5 个跨区域城市（如北上广蓉汉）：搜索、区县柱状图、走势、地图渲染全部正常。
- [ ] 预测功能在新增数据城市上可用（活跃模型正常出预测）。
- [ ] 覆盖统计（城市/区县/快照/地图数量）落入本任务记录，必要时更新 README/文档中的数据说明。
