## 2. 环境配置

### 2.1 环境变量（`.env`）

```env
# 数据库
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/housing_price
DATABASE_URL_SYNC=postgresql://postgres:postgres@localhost:5432/housing_price

# Redis
REDIS_URL=redis://localhost:6379/0

# JWT
JWT_SECRET_KEY=your-secret-key-change-in-production
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=30

# 采集
CRAWL_REQUEST_DELAY_MIN=1.0
CRAWL_REQUEST_DELAY_MAX=3.0
CRAWL_MAX_RETRIES=3

# 应用
APP_ENV=development
DEBUG=true
```

### 2.2 配置管理

使用 `pydantic-settings` 管理配置，支持 `.env` 文件与环境变量：

```python
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    database_url: str
    redis_url: str = "redis://localhost:6379/0"
    jwt_secret_key: str
    jwt_algorithm: str = "HS256"
    jwt_access_token_expire_minutes: int = 30
    app_env: str = "development"
    debug: bool = False

    model_config = {"env_file": ".env"}
```

## 3. 编码规范

### 3.1 后端（Python）

| 规范 | 工具 |
|------|------|
| 代码风格 | ruff（format + lint） |

## 4. Docker

### 4.1 docker-compose.yml（开发环境）

```yaml
services:
  postgres:
    image: postgres:16
    environment:
      POSTGRES_DB: housing_price
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data

  redis:
    image: redis:7
    ports:
      - "6379:6379"

  backend:
    build: ./backend
    command: uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
    ports:
      - "8000:8000"
    env_file: .env
    volumes:
      - ./backend:/app
    depends_on:
      - postgres
      - redis

  frontend:
    build: ./frontend
    command: npm run dev -- --host 0.0.0.0
    ports:
      - "5173:5173"
    volumes:
      - ./frontend:/app
      - /app/node_modules

volumes:
  pgdata:
```

### 4.2 Backend Dockerfile

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 4.3 Frontend Dockerfile

```dockerfile
FROM node:20-slim

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .

CMD ["npm", "run", "dev", "--", "--host", "0.0.0.0"]
```

### 4.4 生产环境（M3-4 已落地）

生产形态由根目录 `docker-compose.prod.yml` 定义，与 dev 栈可并存（独立命名卷，
postgres/redis/backend 不对外发布端口，仅 frontend 发布 `8080:80`）：

- **Frontend**：`frontend/Dockerfile.prod` 多阶段构建（node build → nginx:1.27-alpine），
  `frontend/nginx.conf` 提供 SPA fallback、`/api/` 反代 `backend:8000`、assets 缓存。
- **Backend**：复用 `backend/Dockerfile`，compose 覆盖 command 为
  `alembic upgrade head && uvicorn --workers 2`（启动自动迁移；多 worker 取代
  早期设想的 Gunicorn，uvicorn 原生 `--workers` 已够用）。
- **配置**：`cp .env.prod.example .env.prod` 后替换 `JWT_SECRET_KEY`/`POSTGRES_PASSWORD`；
  `.env.prod` 已被 .gitignore 排除。
- 全部服务带 healthcheck 与 `restart: unless-stopped`。

```bash
# 构建 + 启动
cp .env.prod.example .env.prod   # 首次；改掉密钥
docker compose -f docker-compose.prod.yml --env-file .env.prod up -d --build

# 首次初始化（空库 → schema 已由启动迁移建好）
docker compose -f docker-compose.prod.yml exec backend uv run python scripts/create_admin.py <username> <email> <password>
# 采集入数据：进入容器用 PipelineRunner 跑目标城市，或从已有环境导入 dump

# 访问：http://<host>:8080（前端），API 经 nginx /api 反代
```

## 5. CI/CD（GitHub Actions）

```yaml
# .github/workflows/ci.yml
name: CI

on: [push, pull_request]

jobs:
  backend:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_DB: test_db
          POSTGRES_USER: postgres
          POSTGRES_PASSWORD: postgres
        ports: ["5432:5432"]
      redis:
        image: redis:7
        ports: ["6379:6379"]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install -r backend/requirements.txt -r backend/requirements-dev.txt
      - run: ruff check backend/
      - run: ruff format --check backend/
      - run: mypy backend/app/
      - run: pytest backend/tests/ -v --cov=app --cov-report=term-missing

  frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
      - run: cd frontend && npm ci
      - run: cd frontend && npm run type-check
      - run: cd frontend && npm run lint
      - run: cd frontend && npm run build
```

## 6. 测试策略

| 层级 | 框架 | 覆盖目标 |
|------|------|---------|
| 单元测试 | pytest | 采集解析、清洗逻辑、特征工程、ML 训练/推理 |
| API 测试 | pytest + httpx (AsyncClient) | 所有 REST 端点、鉴权、错误场景 |
| 快照测试 | pytest | 采集适配器用固定 HTML 快照验证解析结果 |
| 前端单元 | Vitest | Store 逻辑、工具函数 |
| 前端组件 | Vue Test Utils | 关键组件渲染与交互 |
| E2E | Playwright | 关键用户路径（搜索→图表→对比→预测） |

## 7. 日志

- **后端**：Python `logging`，JSON 格式输出，级别由 `APP_ENV` 控制（dev=DEBUG, prod=INFO）。
- **采集日志**：除 Python logging 外，关键采集事件持久化到 `crawl_log` 表。
- **前端**：开发环境 console，生产环境仅 error。
