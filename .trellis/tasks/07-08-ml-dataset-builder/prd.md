# ML 数据：多源训练集构建器

## Goal

把 330 城年度挂牌数据（3206 行，当前 ML 死资产）纳入月度训练集：年度点校准
（逐年分段比值）+ 月度插值扩充 + source/口径特征 + 样本降权，同时保持既有
月度序列训练路径不回退。

## Requirements

- R1 新增训练集构建模块，按源分别取序列（不走 `select_merged_snapshots` 的
  单值合并，避免口径混杂拼接）：
  - 月度源（creprice、kaggle_lianjia）：沿用现有连续月序列构建（插值、缺失率门槛）。
  - 年度源（listing_annual_58 等 `granularity=annual`）：年度 12 月点之间线性插值为
    月度序列，全部样本标记 `is_annual_interp` 并降权。
- R2 口径校准：用双口径重叠 (region, month) 对逐年估计「挂牌→成交」比值曲线
  （当前仅北京 8 对），年度挂牌价按所在年比值校准；重叠期外沿用最近年份比值。
  校准曲线与样本对数记入 dataset meta。
- R3 特征扩展：新增 `basis_enc`（listing/transaction）、`is_annual_interp` 两列；
  特征列函数保持向后兼容（旧模型按其 meta.features 推理不受影响）。
- R4 样本权重：年度插值样本权重 < 真实月度样本（初始 0.3，常量可调）；训练器
  透传 sample_weight。
- R5 dataset meta：构建结果附带指纹（各源样本数、区域数、月份范围、行数、
  校准曲线），供训练 meta 记录（对接 ml-train-eval 子任务）。
- R6 同一 (region, month) 若同时有月度真实值与年度插值值，真实月度值优先，
  不产生重复行。

## Acceptance Criteria

- [ ] 用全库数据构建训练集：包含年度扩充样本，区域数远大于现状（现状仅
      creprice 28 区县 + 北京），meta 指纹可见各源分布
- [ ] 北京序列：2010-2018 用 kaggle 真实月度值，2018-2024 缺口由校准后的年度
      插值补齐，无重复月份
- [ ] 逐年比值曲线由重叠对计算得出且落在实证范围（0.79~1.12），重叠期外取
      最近年值
- [ ] 纯月度路径回归：仅用 creprice 城市训练的结果与改造前口径一致（样本数、
      特征含义不回退）
- [ ] 新增单元测试覆盖：年度插值、校准、优先级去重、权重、指纹；全量测试通过

## Notes

- 折价系数单值方案已被实证否定（比值 0.79→1.09 漂移），改逐年分段。
- 年度源识别走 `source_policy.SOURCE_META` 的 granularity/basis，不得在 ML 层
  硬编码源名清单。
