# 数据质量校验框架

## Goal

"尽一切可能保证训练数据有效性与真实性"的机制化落地：进入库的数据先过校验，
已在库的数据可被跨源审计，模型与数据的新鲜度可被检查。

## Requirements

- R1 导入校验器（共享模块）：所有快照写入路径（creprice 管线、年度导入、
  未来新源）统一过 `snapshot_validator`——
  a) 单值域：¥/㎡ ∈ [500, 200000]，超界跳过并计数；
  b) 跳变检测：同区域环比 |Δ| > 40% 标记（不拦截，进审计报告）；
  c) 空值/格式：year_month 格式、region 存在性（既有逻辑保持）。
  校验结果计入 job 结果（imported/rejected/flagged）。
- R2 跨源一致性审计：`GET /admin/data-quality/report`（admin）返回 JSON 报告：
  a) 多源重叠区域的月度水平比值分布与离群清单（比值超出 [0.5, 2.0] 的
     region-month）；
  b) creprice 环比方向 vs NBS 二手指数环比方向的一致率（70 城重叠月）；
  c) 58 年度同比方向 vs 指数年累计方向一致率；
  d) 各源覆盖/新鲜度（区域数、最新月、距今月数）。
- R3 模型新鲜度：报告含当前库数据指纹 vs 活跃模型 meta.dataset.fingerprint
  对比（一致=模型新鲜；不一致=建议重训）。
- R4 报告落地：管理页 DataManageView 或 ModelManageView 加「数据质量」入口
  展示报告要点（离群数、方向一致率、模型新鲜度徽标）；表格粒度明细可只走
  API/JSON。
- R5 实操收尾：跑一次真实审计，报告结论写入任务留档；如发现离群数据，
  给出处理建议（不在本任务内清数）。

## Acceptance Criteria

- [ ] 构造超界/跳变数据的导入被正确拦截/标记，job 结果可见计数
- [ ] 审计报告四节齐全，真实库上可产出；方向一致率有数值
- [ ] 模型新鲜度判定正确（重训前 stale、重训后 fresh）
- [ ] 前端可见质量入口与要点；全量 pytest + 前端 build 通过
- [ ] 真实审计结论留档（发现的离群/不一致清单与建议）

## Notes

- 轻量子任务：无迁移、无新表（报告即时计算+短 TTL 缓存可选）；design 并入
  implement.md。
- 依赖 nbs-index-import 的指数数据（R2 b/c）；若指数未导入，对应小节返回
  "no index data" 而非报错。
- 跳变阈值 40%、比值域 [0.5,2.0] 为初始常量，均集中定义可调。
