# 中介站（链家 / 贝壳 / 安居客）作为 ¥/㎡ 城市·商圈均价采集源 —— 反爬可行性研究

> 研究日期：2026-07-08。实测方法：curl（直连 vs 代理）+ Playwright 真机浏览器 + GitHub API + web_search。
> 目标数据形状：城市 → 区县 / 商圈（板块）月度二手房挂牌 ¥/㎡ 均价（`PriceSnapshot`）。

---

## 0. 结论速览（TL;DR）

| 站点 | 目标数据能否拿到 | 反爬强度 | 现成可用 spider | 本机（数据中心 IP）实测 | 综合可行性 |
|---|---|---|---|---|---|
| **链家 lianjia.com** | 能（需自聚合 `/xiaoqu/` 版块页） | 中-高（IP+行为封禁，硬 Forbidden，无验证码可解） | ❌ 无（经典库已删，fork 全为 2018-2019 陈旧） | 首页/列表第 1 页可开，第 2 页起 302，~15 次后整机 IP 被硬封 | **中低** |
| **贝壳 ke.com** | 能（与链家同架构） | 中-高（与链家同一套反爬） | ❌ 同上 | 与链家完全一致 | **中低** |
| **安居客 anjuke.com** | 能（`/fangjia/` 直接给板块均价，形状最好） | **高**（58 antibot + 网易易盾滑块，首次请求即验证码） | ❌ 仅有滑块破解库，无数据 spider | 一进就跳 `callback.58.com/antibot/verifycode` 滑块 | **低** |

**一句话判定**：三站数据形状都比 creprice 更贴合我们的模型，但**在本项目当前的出口网络（数据中心 IP，且给定代理直接连不通中介站）下，三站均无法开箱采集**。要落地必须投入「国内住宅代理池 + 真机浏览器 + 严格限速」，其中安居客还要额外破滑块。**没有任何一个现成 GitHub 项目今天可直接跑通商圈均价采集**——最经典的 `jumper2014/lianjia-beike-spider` 已被删除（404），所有 fork 都停留在 2018-2019、早于链家现行反爬。

**推荐**：若一定要做中介源，选 **贝壳/链家的 `/xiaoqu/` 版块页聚合**路径（无验证码、只是 IP 封禁，比安居客滑块省事），预估稳定性 **中低**、高维护。**放弃安居客**做主源（滑块+易盾成本最高）。但更务实的建议：相对 creprice，中介站不是「更省力」的替代而是「更贵」的备选，优先级应低于其它半官方聚合源（见 §6）。

## 1. 环境 / 网络前提实测（关键背景）

- **本机出口 IP = `54.37.83.196`**（OVH 数据中心，欧洲区段）。此 IP 由安居客反爬页与链家封禁页共同暴露（安居客页面标题 `请输入验证码 ws:54.37.83.196`、正文「访问过于频繁…（54.37.83.196）」）。**即本机不是国内住宅 IP，而是海外机房 IP**——这是三站快速封禁的根本原因。
- **给定代理不可用**：`http://…@10.0.0.1:2260`，出口 IP 实测 `103.62.49.170`。
  - `curl -x proxy https://api.ipify.org` → HTTP 200（代理本身是活的）。
  - `curl -x proxy` 访问链家 / 贝壳 / 安居客 → **全部 HTTP 000（连接失败/被 reset）**。
  - 结论：**该代理对三家中介站完全连不通**（比直连还差），无论美国还是该 IP 段都被风控/拒绝。**代理路径直接排除**。
- 直连（机房 IP）TCP 层可达三站（GFW 未拦这类商业站），但**应用层反爬在数据中心 IP 上极快触发**。

## 2. 逐站实测详情

### 2.1 链家 lianjia.com

**(1) 目标数据位置**
- `https://bj.lianjia.com/ershoufang/`（二手房列表）：**每套房源含单价**，HTML 选择器 `div.unitPrice > span`，且带 `data-price` 属性，例如：
  `<div class="unitPrice" data-price="47848"><span>47,848</span>元/平...</div>`，每页 30 套。
- 商圈/区县均价的正确来源是**小区聚合页** `/xiaoqu/{区县}/{商圈拼音}/`：列出该板块各小区均价，老 spider 就是抓这里再聚合成「版块均价」。**这才是贴合我们商圈均价模型的数据位置**。
- `/ershoufang/{区县}/`、`/ershoufang/{商圈}/` 过滤列表页也带聚合信息。
- **`/fangjia/`（官方均价专页）：直连 302 → 跳回首页**（`bj.lianjia.com/fangjia/` 与 `/fangjia/chaoyang/` 均 302 到 `bj.lianjia.com/`，页面无均价数据）。**该专页对匿名/无 JS 客户端不可用**。

**(2) 反爬实测（决定性）**
- 干净 IP 首次：`/ershoufang/` 第 1 页 = **HTTP 200**（188 KB 真数据）。
- **第 2 页起全部 302**：`pg2/pg3/pg4/pg5` = 302 size 0（跳回首页）。区县页 `/ershoufang/chaoyang/`、商圈页均 302。
- **带 Cookie 也无解**：先访问首页拿到 `lianjia_uuid / lianjia_ssid / select_city`，再带 Cookie+Referer 请求 pg2 → **仍 302**。在全新城市（gz）走「首页→列表→等 3s→pg2」标准会话流，**pg2 依旧 302**。
- **~15-20 次请求后整机 IP 被硬封**：连第 1 页也 302。Playwright 真机访问 `/ershoufang/chaoyang/` → 直接跳 **`hip.lianjia.com/forbidden?...reason=抱歉，您的请求未能通过我们的安全检查`**。
- **无验证码可解**——链家不给滑块，直接 Forbidden。封禁基于 **IP + 行为评分**，不是请求签名（第 1 页无加密参数即可访问，说明拦截在服务端行为层，不在参数层）。

**判定**：技术上可行（真机+住宅IP+限速可抓 `/xiaoqu/` 版块页聚合），但**机房 IP 秒封、无现成可用库、维护成本高**。

### 2.2 贝壳 ke.com

- 贝壳与链家同属一家，**反爬完全一致**：`bj.ke.com/ershoufang/` 第 1 页 = HTTP 200（231 KB，含 `unitPrice`）；**pg2 = 302；区县页 chaoyang = 302**。
- 数据形状、选择器、`/xiaoqu/` 聚合逻辑与链家同构。
- **判定**：与链家同级（中低）。二选一时贝壳/链家等价，随便挑一个。

### 2.3 安居客 anjuke.com（58 同城旗下）

**(1) 目标数据位置（形状最好）**
- `https://www.anjuke.com/fangjia/{城市拼音}{年份}/`（如 `/fangjia/beijing2024/`）：历史上**直接给出城市/区域/板块均价 + 环比同比**，无需自聚合，最贴合我们的模型。
- 但——**该页在机房 IP 上完全打不开**。

**(2) 反爬实测**
- 首次直连即 302 → **`https://www.anjuke.com/esfcommon-captcha-wy?...&from=antispam`**（`-wy` = 网易易盾）。
- `/fangjia/…` 直连 302 → 同上 antispam captcha；Playwright 真机 → 跳 **`https://callback.58.com/antibot/verifycode?...namespace=cloud_anjuke_fangjia_pc`**，页面：「访问过于频繁，本次访问做以下验证码校验（54.37.83.196）请在五分钟内完成验证 [点击按钮进行验证]」——点击后进入**滑块验证**。
- 用了两套商业反爬：**58 同城 antibot** + **网易易盾（Yidun）滑块**。首次请求即触发，比链家激进得多。

**判定**：数据形状最好，但反爬最重（滑块+易盾+机房IP三重障碍）。**不建议作为主源**。

## 4. 绕过手段可行性评估

| 障碍 | 有无开源方案 | 在本机（机房 IP）能否用 | 诚实判断 |
|---|---|---|---|
| 链家/贝壳 IP+行为封禁 | **无验证码可破**——只能换 IP + 真机 + 限速 | ❌ 机房 IP 秒封，给定代理连不通 | 需**国内住宅代理池**（芝麻/快代理/青果等付费）+ Playwright/stealth + 严格限速。playwright-stealth 只帮过指纹，**IP 才是决定因素** |
| 安居客 58 antibot + 网易易盾滑块 | SliderCracker（JS逆向）、ddddocr（缺口OCR）+ 轨迹模拟 | ❌ 机房 IP 首次即滑块 | 技术上可破，但易盾频繁更新、需持续逆向；**且仍要住宅 IP**。ROI 低 |

| **美国/海外 IP 直接可用性** | — | ❌ | **明确结论：海外机房 IP 下三站基本不可用**。链家硬 Forbidden、安居客滑块。给定代理（103.62.49.170）对三站 HTTP 000。**必须国内住宅代理** |

**核心瓶颈是「国内住宅出口 IP」而非验证码**。没有住宅 IP，破了滑块也没用（链家甚至不给你滑块解）。

## 5. 最小可行路径（若必须实现一个中介源）

**首选：贝壳 ke.com（或链家）的 `/xiaoqu/{区县}/{商圈}/` 版块页聚合**
- 手段：Playwright 真机（headful + stealth）+ **国内住宅代理池（按城市轮换 IP）** + 严格限速（每 IP 每分钟个位数请求、随机延时）+ Cookie 会话维持。
- 流程：遍历城市→区县→商圈 的 `/xiaoqu/` 页，取各小区均价 → 聚合成商圈 ¥/㎡ 均价 → 写 `PriceSnapshot`。
- **优势**：无验证码（只需绕过 IP 封禁），比安居客省一整层滑块工程。
- **预估稳定性：中低**。依赖付费住宅代理，链家风控会持续调整，需要长期维护（IP 池健康度、限速调参、选择器随改版失效）。

**次选/放弃：安居客** —— 数据形状最好（`/fangjia/` 直出板块均价）但滑块+易盾+住宅IP 三重成本，**不建议作为主源**，除非已具备住宅代理且愿意投入滑块破解维护。

## 6. 最终推荐

1. **实现哪个**：如果多源采集里必须放一个中介源，**做贝壳/链家 `/xiaoqu/` 版块均价聚合**（Playwright + 国内住宅代理 + 限速），稳定性定位「中低」，并明确标注为高维护、需付费代理。
2. **放弃哪个**：**安居客不做主源**（滑块+网易易盾成本最高，ROI 最低）；如果哪天需要板块均价的「现成形状」，再评估投入其滑块破解。
3. **对本项目的务实建议**：在**当前出口网络（海外机房 IP + 给定代理连不通中介站）下，三站均无法开箱采集，且无现成可用开源库**。相对已被限流的 creprice，中介站**不是「更省力」的替代，而是「更贵、更易碎」的备选**。建议：
   - **短期**：优先给 creprice 换/加国内住宅 IP + 限速，成本远低于新做中介源；
   - **中期**：评估其它半官方聚合源（国家统计局 70 城指数、各地住建/房管局网签数据、贝壳研究院/中指院报告、「中国房价行情」平台），数据形状同样是城市/区域均价且反爬远轻；
   - 把中介站直采列为**最后手段**，仅在拿到稳定国内住宅代理池后再启动，且首选贝壳 `/xiaoqu/` 路径。
