# creprice 主数据源方针落地（父任务）

## 背景与方针（2026-07-08 grilling 会话定稿）

用户决定：**训练与预测只认 creprice 采集数据；其他源（58 年度挂牌、kaggle
链家、NBS 指数）仅作展示对比，全站按源硬隔离，不再跨源合并。**

动机：多源合并展示造成"假月度数据"观感（如泉州：creprice 只有 2025-07 起，
更早月份被 58 年度数据顶上来）；训练混入年度插值样本使指标与预测名不副实。
单一来源的数据口径一致，可用性更有保障；creprice 覆盖靠持续采集扩张。

## 已定决策（grilling 共识，新会话执行时不要重新讨论）

1. **接受覆盖塌缩**：预测从 330+ 城退到 creprice 实际覆盖（当前 5 城/40 区域，
   481 条月度快照），随采集扩张自然增长。
2. **不 git revert**：多源导入能力保留（展示要用），ML 侧用训练源白名单挡住，
   代码路径可逆。
3. **全局源切换器（方案 A）**：顶栏单选 creprice / 58 年度 / kaggle / NBS 指数，
   默认 creprice；全站单源渲染；**彻底删除 `select_merged_snapshots` 合并路径**。
4. **缓训 + 旧模型全删**：不立即重训；删除全部现有模型（v1.7 等"没有参考意义"），
   预测功能进入空窗期（前端显式"数据积累中"），等全量爬取完成后再训 v1.8。
5. **全量首采主动推进**：沿用 07-07-full-data-crawl（保留，07-07-admin-platform
   下），管理端分批 20-50 城串行限速爬完数百城；每日调度只做增量维护。
   爬完是 v1.8 重训的触发信号（届时再开重训任务，本父任务不含）。
6. **数据不删**：58 年度 3206 条、kaggle 82 条、NBS 指数照留（展示素材）；
   要清的只有 `prediction` 表旧预测行与 `backend/models/` 全部模型文件。
