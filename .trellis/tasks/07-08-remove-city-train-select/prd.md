# 去掉指定城市训练功能

## Goal

移除训练接口和前端页面中"指定城市训练"的功能，只保留全量训练模式。

## Background

每个城市仅 13 条月度数据，按城市子集训练的模型质量远不如全量训练。该功能给用户造成"定制化模型"的错觉，实际效果差。全量训练 + city_tier 特征已足以区分不同城市的规律。

## Requirements

### R1: 后端 API 变更
- `TrainRequest` schema 中移除 `city_codes` 字段（或忽略传入值）
- 训练逻辑始终使用全部城市数据
- API 文档更新

### R2: 前端 UI 变更
- 训练页面移除城市选择控件（多选框/下拉等）
- 训练按钮直接触发全量训练
- 如有相关提示文案需更新

### R3: 向后兼容
- 旧版请求体中如果传了 city_codes 字段，不报错，静默忽略即可

## Acceptance Criteria

- [ ] 训练 API 不再接受 city_codes 参数影响训练范围
- [ ] 前端训练页面无城市选择控件
- [ ] 训练始终使用全量数据（可通过日志验证加载了所有城市）
- [ ] 旧格式请求不报错
- [ ] 现有测试通过（相关测试用例需更新）
