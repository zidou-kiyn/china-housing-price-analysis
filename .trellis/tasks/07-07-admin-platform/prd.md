# 管理端完善与全量数据平台（父任务）

## Goal

将系统从「命令行驱动的 3 城演示」升级为「管理端自助运营的全国数据平台」：部署配置收敛安全、管理员可在前端完成用户管理 / 数据采集 / 地图爬取 / 模型训练与选择，并最终完成全国城市数据与地图的全量爬取。

## 源需求（用户原始 5 条，2026-07-07）

1. docker compose 不管生产还是 dev 环境都不暴露数据库、Redis 端口；后端如无必要也不暴露；volume 使用 `./` 目录；合并 prod 配置只保留一份。
2. 完善管理端用户管理功能。
3. 给管理员配置好在前端进行数据爬取和地图爬取的功能。
4. 爬取完全部的城市和地图数据。
5. 给管理员配置好模型选择、模型训练等功能。

## 已确认决策（2026-07-07 与用户确认）

- **dev 全容器化**：backend/frontend 也进 compose，DB/Redis/backend 端口彻底不暴露，仅前端暴露端口；脚本与 pytest 改为 `docker compose exec` 执行。
- **compose 合并策略**：`docker-compose.yml`（生产安全基准）+ `docker-compose.override.yml`（dev 差异，自动叠加），删除 `docker-compose.prod.yml`。
- **GeoJSON 改由后端存储 + API 提供**：不再写 `frontend/public/geo/`，前端改为从 API 加载地图，保证生产环境管理端爬图即时生效。

## 任务地图（执行顺序）

| # | 子任务 | 交付物 | 依赖 |
|---|--------|--------|------|
| 1 | 07-07-compose-unify | 单一 compose 基准 + dev override，端口收敛，`./` 卷 | 无 |
| 2 | 07-07-admin-user-mgmt | 封禁/启用、删除、搜索筛选（前后端） | 无 |
| 3 | 07-07-admin-data-collect | 后台任务机制 + 采集/地图爬取 API + GeoJSON 服务化 + 数据管理页 | 无（复杂，核心） |
| 4 | 07-07-admin-model-mgmt | 训练改后台任务 + 模型管理页 | #3 的后台任务执行器 |
| 5 | 07-07-full-data-crawl | 全国城市数据 + 地图全量落地（执行型） | #3（用其功能执行，兼做验收） |

## 跨子任务验收标准（父任务集成验收）

- [ ] `docker compose up -d` 一条命令起完整 dev 栈；`docker compose -f docker-compose.yml up -d` 起生产栈；`docker compose ps` 中 postgres/redis/backend 无任何宿主机端口映射。
- [ ] 管理员登录前端后，通过导航可达：用户管理、数据管理、模型管理 三个管理页。
- [ ] 在数据管理页触发一个城市的采集与地图爬取，任务进度可见，完成后该城市在普通用户端可搜索、图表与地图正常展示（dev 与 prod 行为一致）。
- [ ] 在模型管理页触发一次训练，训练异步执行、进度可查，完成后可切换活跃模型且预测接口生效。
- [ ] creprice 全量城市数据与对应 GeoJSON 地图爬取完成，任选 3 个非闽城市抽查前端展示正常。
- [ ] 现有测试套件（后端单测、前端 E2E）在新运行方式下全部通过。

## 边界

- 不引入 Celery/APScheduler 等重量级任务框架，采用进程内 asyncio + 数据库任务表的轻量方案。
- 不做定时调度（cron 式自动采集），仅管理员手动触发。
- 数据源仍仅 creprice（M2-1 链家/安居客已确认暂缓）。
